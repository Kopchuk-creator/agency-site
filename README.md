# Сайт агенції — макет

Готовий фронтенд-макет за ТЗ. Усі 11 екранів, адаптив, анімації, валідація форм.
**Це макет для погодження, а не робочий сайт:** заявки нікуди не надсилаються,
контакти й реквізити — заглушки, індексація закрита.

## Як подивитись

```bash
npx serve -l 4321 .
```

Далі відкрити `http://localhost:4321`. Збірка не потрібна — чистий HTML/CSS/JS.

## Публікація на GitHub Pages

Репозиторій → **Settings → Pages → Source: Deploy from a branch → main / (root)**.
За хвилину сайт буде на `https://<нік>.github.io/<репозиторій>/`.

Усі шляхи всередині — **відносні**, тож сайт однаково працює і в корені домену,
і в підпапці. Не замінюйте їх на абсолютні (`/styles.css`) — на Pages це ламає стилі.

Файл `.nojekyll` вимикає обробку Jekyll: вона не потрібна і лише сповільнює деплой.

**Сайт закритий від індексації** (`noindex` на всіх сторінках) — поки це макет.
Перед справжнім запуском див. чек-лист нижче.

## Структура

```
index.html                              головна, 11 екранів
styles.css                              уся стилістика + дизайн-токени
main.js                                 інтерактив: вітрина, таби, форма, меню
analytics.js                            GA4 / GTM / Meta / Ads — вимкнено, поки ID порожні
blog/index.html                         блог
blog/yak-obraty-pidryadnyka/            стаття-зразок
polityka-konfidentsiynosti/             політика конфіденційності (шаблон)
publichna-oferta/                       публічна оферта (шаблон)
```

# Чек-лист переведення в робочий сайт

## 1. Прибрати режим макета

- [ ] Видалити блок `<div class="demo-bar">` з `index.html`
- [ ] Видалити правило `.demo-bar` зі `styles.css` (секція «ПОЗНАЧКА МАКЕТА»)
- [ ] Прибрати `<meta name="robots" content="noindex, nofollow">` з **усіх 5 сторінок**
- [ ] Розкоментувати `canonical` і `og:image`, підставити реальний домен
- [ ] У `main.js` повернути справжній текст підтвердження форми (позначено коментарем)

## 2. Підставити реальні дані

- [ ] Назва агенції — зараз скрізь «Агенція»
- [ ] Телефон — зараз `+38 (067) 000-00-00`
- [ ] Email — зараз `hello@agency.ua`
- [ ] Посилання на Telegram, Viber, Instagram, LinkedIn, Behance
- [ ] Реквізити ФОП у футері та в оферті: ПІБ, РНОКПП, IBAN
- [ ] Ім'я та роль партнера (зараз «Ім'я партнера») у секції «Команда»
- [ ] Фото команди 3:4 замість плейсхолдерів (`.member__photo`)
- [ ] Картинка для соцмереж `og.jpg` (1200×630)

## 3. Юридичні тексти

- [ ] **Узгодити політику конфіденційності та оферту з юристом** — зараз це шаблони
- [ ] Перевірити, що умови оферти збігаються з тим, що обіцяє головна
      (50/50, гарантія 6 міс, 2 раунди правок)

## 4. Підключити бекенд

- [ ] Ендпоінт для форми аудиту — точка інтеграції позначена в `main.js`
      (шукати «Точка інтеграції з бекендом»)
- [ ] Пересилання заявок: CRM / Google Sheets + email + Telegram
- [ ] Генерація PDF-звіту аудиту
- [ ] Календар: замінити `https://cal.com/` на свій профіль Cal.com або Calendly
- [ ] Онлайн-чат або віджет зворотного дзвінка (зараз кнопка веде на форму)

## 5. Аналітика

Відкрити `analytics.js`, вписати ID у `CONFIG`:

- [ ] `ga4` — Google Analytics 4 (`G-XXXXXXXXXX`)
- [ ] `gtm` — Google Tag Manager (`GTM-XXXXXXX`)
- [ ] `metaPixel` — Meta Pixel (15–16 цифр)
- [ ] `googleAds` — Google Ads (`AW-XXXXXXXXX`)

Поки поле порожнє — відповідний сервіс не вантажиться взагалі.
Подія конверсії `generate_lead` / `Lead` уже підключена до форми.

## 6. Контент

- [ ] Кейси реальних клієнтів — місце позначено коментарем у секції 6.
      Формат за ТЗ: скріншоти, задача, що зробили, результат у цифрах, відгук
- [ ] Замінити відгуки на справжні (зараз заглушки з вигаданими іменами)
- [ ] Блок «5,0 · 12 відгуків у Google» — підставити реальні дані або прибрати
- [ ] Логотипи клієнтів у стрічці
- [ ] Дві статті блогу ведуть на неіснуючі сторінки — написати або прибрати картки
- [ ] Демо під ніші без готового сайту (6 з 10 позначені «Демо готуємо»)

## 7. Перед публікацією

- [ ] Перевірити на реальному телефоні, не тільки в DevTools
- [ ] Прогнати PageSpeed Insights на бойовому домені
- [ ] `sitemap.xml` і `robots.txt`
- [ ] Підключити Cloudflare (CDN + кеш)
- [ ] Перевірити, що форма реально доставляє заявку

## Технічні нотатки

**Стек.** Vanilla HTML/CSS/JS без збірки — ТЗ це дозволяє
(«або те, що зручніше — головне швидкість і SEO»). Дає LCP 128 мс і нульову
вартість підтримки. Якщо знадобиться CMS чи багато сторінок — переносити на Next.js.

**Анімації.** Усі поважають `prefers-reduced-motion`. Виняток — стрічка логотипів:
вона сповільнюється вдвічі, але не зупиняється, і має паузу при наведенні (WCAG 2.2.2).
Якщо у вас вимкнені анімації в системі — решта ефектів буде приглушена, це навмисно.

**Превʼю сайтів.** Вітрина в героєві та картки робіт — не картинки, а CSS-макети
зі справжніми палітрами демо-сайтів. Нуль зовнішніх запитів, тому не б'ють по швидкості.

**Іконки.** Інлайновий SVG, без бібліотек.
